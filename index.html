¿Qué es FTP?

Imagina un Buzón Digital universal. FTP (File Transfer Protocol) define cómo dos sistemas intercambian archivos de forma ordenada, confiable y estandarizada.

FTP separa el control (comandos) del canal de datos. Se usa para listar, subir y bajar archivos entre un cliente y un servidor remoto.

Nota: FTP fue estandarizado en 1971 (RFC 959) y sigue siendo fundamental en la transferencia de datos, aunque en entornos modernos se prefiere SFTP por seguridad.

Metáfora del Buzón

Ordenado, jerárquico y persistente — así piensa FTP sobre archivos y directorios remotos.

Modos de Conexión

Modo Activo

Conexión Activa

El cliente establece el puerto de escucha y el servidor inicia la conexión de datos hacia él.

  • Cliente escucha en puerto > 1024
  • Servidor conecta desde puerto 20
  • Requiere firewall permisivo en cliente
  • Histórico, menos común hoy
Modo Pasivo

Conexión Pasiva

El servidor abre un puerto aleatorio y el cliente se conecta para recibir datos.

  • Servidor determina puerto de datos
  • Cliente inicia ambas conexiones
  • Mejor para clientes detrás de NAT
  • Estándar moderno actual

Tipos de Transferencia

ASCII

Modo Texto

Convierte saltos de línea según el sistema operativo destino (LF → CRLF en Windows, etc.).

  • Archivos de texto (.txt, .html)
  • Fuente de código (.py, .js)
  • Puede corromper binarios
  • Comando: TYPE A
BINARIO

Modo Binario

Transferencia byte-por-byte sin conversión. Seguro para cualquier tipo de archivo.

  • Imágenes (.jpg, .png)
  • Ejecutables (.exe, .zip)
  • Audio/Vídeo (.mp3, .mp4)
  • Comando: TYPE I

La evolución de la seguridad

El FTP clásico transmite credenciales en texto plano; esto es inseguro en redes públicas.

FTP Classic

Sin cifrado — vulnerable a ataques 'Man-in-the-Middle'.

SFTP (SSH)

Todo el tráfico viaja por un túnel SSH cifrado — recomendado para producción.

Alternativas Modernas

FTP sigue siendo usado, pero existen protocolos más seguros y eficientes:

SFTP (SSH File Transfer)

✓ Recomendado — Todo cifrado, autenticación SSH, puerto 22. Estándar actual para producción.

FTPS (FTP sobre SSL/TLS)

FTP con cifrado SSL/TLS, puerto 990. Requiere certificados, menos común que SFTP.

SCP (Secure Copy)

Basado en SSH, sintaxis simple. Ideal para copias puntuales entre servidores.

rsync

Sincronización bidireccional, solo copia cambios. Excelente para backups y sincronización.

HTTP/HTTPS

Descargas web, muy eficiente en redes públicas. No requiere credenciales FTP.

AWS S3 / Cloud Storage

Escalable, seguro, APIs modernas. Solución cloud para almacenamiento distribuido.

Preguntas Frecuentes

¿Cuál es la diferencia entre FTP activo y pasivo?

En modo activo, el servidor abre la conexión de datos; en pasivo, el cliente. Pasivo es más compatible con firewalls modernos y es el estándar actual.

¿FTP es seguro?

FTP clásico NO es seguro — transmite credenciales en texto plano. Usa SFTP o FTPS en entornos de producción. Para datos públicos o dentro de redes de confianza, puede ser aceptable.

¿Qué cliente FTP debo usar?

Opciones populares: FileZilla (GUI, multiplataforma), WinSCP (Windows), Cyberduck (Mac), o línea de comandos con ftp o sftp.

¿Por qué FTP aún se usa si SFTP es mejor?

Compatibilidad heredada. Muchos servidores antiguos, dispositivos IoT y sistemas embebidos solo soportan FTP. Además, algunos ISP ofrecen hosting solo con FTP.

¿Cuál es el máximo tamaño de archivo que puedo transferir?

FTP teóricamente no tiene límite, pero depende del servidor (configuración, espacio en disco, timeouts). Archivos > 5GB pueden requerir configuración especial.
